**What is a Hash Table and How Does It Work?**

A **hash table** (also known as a hash map) is a data structure that stores key-value pairs. It uses a **hash function** to compute an index, also called a hash code or hash, into an array of **buckets** or **slots**. This index is used to find the desired value associated with a given key. Hash tables are highly efficient for operations like insertion, deletion, and search, often achieving an average time complexity of O(1) (constant time). (Confirmed)

**Core Components:**

1.  **Keys:** Unique identifiers for each data element. Keys are used to locate their associated values. (Confirmed)
2.  **Values:** The data associated with each key. This can be any type of information. (Confirmed)
3.  **Hash Function:** A mathematical or algorithmic function that takes a key as input and returns an integer hash code. A good hash function distributes keys uniformly across the array to minimize collisions. (Confirmed)
    *   **Properties of a good hash function:**
        *   **Deterministic:** Always produces the same output for the same input. (Confirmed)
        *   **Efficient:** Fast to compute. (Confirmed)
        *   **Uniform Distribution:** Maps keys evenly across the available buckets. (Confirmed)
        *   **Avalanche Effect:** A small change in the key should ideally result in a significant change in the hash code. (Confirmed)
4.  **Buckets (or Slots):** An array where the key-value pairs are stored. The hash function determines which bucket a key-value pair belongs to. (Confirmed)

**How Operations Work:**

*   **Insertion (Adding a Key-Value Pair):**
    1.  Take the **key** of the pair to be inserted. (Confirmed)
    2.  Pass the key through the **hash function** to compute a hash code. (Confirmed)
    3.  Convert the hash code into an index within the bounds of the **bucket array** (often using the modulo operator: `index = hash_code % array_size`). (Confirmed)
    4.  Store the key-value pair in the determined bucket. If the bucket is already occupied (a collision), a collision resolution strategy is employed. (Confirmed)

*   **Retrieval (Finding a Value by Key):**
    1.  Take the **key** for which the value is sought. (Confirmed)
    2.  Pass the key through the **hash function** to compute its hash code. (Confirmed)
    3.  Convert the hash code to an index within the bucket array. (Confirmed)
    4.  Go to the determined bucket. If the bucket is empty, the key is not in the hash table. (Confirmed)
    5.  If the bucket contains one or more items, check them to find the one whose key matches the search key. (Confirmed)
    6.  If a match is found, return its associated value. If no match is found after checking all items in the bucket (or if the bucket is empty), the key is not present. (Confirmed)

*   **Deletion (Removing a Key-Value Pair):**
    1.  Follow steps 1-3 for retrieval to locate the bucket containing the key-value pair. (Confirmed)
    2.  If the key is found in the bucket, remove the key-value pair. (Confirmed)
    3.  If the key is not found, no action is needed. (Confirmed)

**Collision Handling:**

A **collision** occurs when two different keys produce the same hash code or map to the same bucket index. Since a bucket can only hold a limited amount of data (or a single item if not using specific collision strategies), collisions must be handled to ensure all data can be stored and retrieved. (Confirmed)

Common collision handling strategies include:

1.  **Separate Chaining:**
    *   **Description:** Each bucket in the array stores a pointer to a secondary data structure (most commonly a linked list, but also potentially a balanced tree or another hash table). When a collision occurs, the new key-value pair is added to the secondary structure associated with that bucket. (Confirmed)
    *   **Pros:** Relatively simple to implement, good performance even with high load factors. (Confirmed)
    *   **Cons:** Can lead to performance degradation if linked lists become very long, requires extra memory for pointers. (Confirmed)

2.  **Open Addressing (or Closed Hashing):**
    *   **Description:** All key-value pairs are stored directly within the bucket array itself. When a collision occurs, the algorithm probes for the next available empty slot in the array according to a defined probing sequence. (Confirmed)
    *   **Probing Sequences:**
        *   **Linear Probing:** Check the next slot (`index + 1`), then `index + 2`, and so on, wrapping around the array. (Confirmed)
        *   **Quadratic Probing:** Check slots `index + 1^2`, `index + 2^2`, `index + 3^2`, etc., wrapping around. (Confirmed)
        *   **Double Hashing:** Use a second hash function to determine the step size for probing. (Confirmed)
    *   **Pros:** No need for extra pointers, potentially better cache performance. (Confirmed)
    *   **Cons:** Can lead to clustering (primary for linear probing), requires careful management of deletion (using tombstone markers), performance degrades significantly as the table fills up. (Confirmed)

**Key Benefits of Hash Tables:**

*   **Fast Average-Case Performance:** For insertion, deletion, and search operations, hash tables offer an average time complexity of O(1). (Confirmed)
*   **Efficient Lookups:** Ideal for scenarios where quick retrieval of data based on a key is paramount. (Confirmed)
*   **Flexibility:** Can store various types of keys and values. (Confirmed)

**Challenges:**

*   **Hash Function Quality:** The performance of a hash table is heavily dependent on the quality of its hash function. A poorly designed hash function can lead to frequent collisions and degrade performance to O(n) in the worst case. (Confirmed)
*   **Collision Resolution:** Implementing an efficient collision resolution strategy is crucial. (Confirmed)
*   **Resizing:** As more elements are added, the hash table may need to be resized (rehashing all elements into a larger array) to maintain performance, which can be an expensive operation. (Confirmed)

**What is Recursion in Programming?**

**Recursion** is a programming technique where a function solves a problem by calling itself with a smaller or simpler version of the same problem. This process continues until a specific condition, known as the **base case**, is met, at which point the function stops calling itself and returns a result. (Confirmed)

**Core Components of Recursion:**

1.  **Base Case:**
    *   This is the condition under which the recursive function stops calling itself. (Confirmed)
    *   It represents the simplest possible instance of the problem that can be solved directly without further recursion. (Confirmed)
    *   Without a base case, a recursive function would call itself infinitely, leading to a stack overflow error. (Confirmed)

2.  **Recursive Step:**
    *   This is the part of the function where it calls itself. (Confirmed)
    *   The recursive call must operate on a modified input that moves the problem closer to the base case. (Confirmed)
    *   This step breaks down the larger problem into smaller, identical subproblems. (Confirmed)

**How It Works (Analogy):**
Imagine you need to count the number of steps to reach the top of a staircase.
*   **Base Case:** If you are already at the top step (or there are no steps left), you stop. (Confirmed)
*   **Recursive Step:** If you are not at the top, you take one step and then ask yourself to count the remaining steps from your new position. You repeat this until you reach the top. (Confirmed)

**Examples:**

*   **Factorial Calculation (n!)**
    *   **Definition:** `n! = n * (n-1) * (n-2) * ... * 1`
    *   **Base Case:** `0! = 1` or `1! = 1`
    *   **Recursive Step:** `n! = n * (n-1)!`

    ```python
    def factorial(n):
        # Base case
        if n == 0 or n == 1:
            return 1
        # Recursive step
        else:
            return n * factorial(n - 1)
    ```
    (Confirmed)

*   **Fibonacci Sequence (F(n))**
    *   **Definition:** `F(0) = 0`, `F(1) = 1`, `F(n) = F(n-1) + F(n-2)` for `n > 1`
    *   **Base Cases:** `F(0) = 0`, `F(1) = 1`
    *   **Recursive Step:** `F(n) = F(n-1) + F(n-2)`

    ```python
    def fibonacci(n):
        # Base cases
        if n <= 0:
            return 0
        elif n == 1:
            return 1
        # Recursive step
        else:
            return fibonacci(n - 1) + fibonacci(n - 2)
    ```
    (Confirmed)
    *   *Note:* This implementation is a clear illustration but is computationally inefficient due to repeated calculations. Techniques like memoization or dynamic programming can optimize this. (Confirmed)

**Advantages of Recursion:**

*   **Elegance and Readability:** For problems that are naturally recursive (like tree traversals, fractals, or certain mathematical functions), recursive solutions can be more concise and easier to understand than iterative ones. (Confirmed)
*   **Simplifies Complex Problems:** Can break down complex tasks into manageable, smaller subproblems. (Confirmed)

**Disadvantages of Recursion:**

*   **Stack Overflow:** Each recursive call adds a new frame to the call stack. If the recursion depth is too large, it can exhaust the stack memory, leading to a stack overflow error. (Confirmed)
*   **Performance Overhead:** Function calls have overhead (setting up stack frames, passing arguments). In some cases, an iterative solution can be more efficient due to less overhead. (Confirmed)
*   **Redundant Computations:** Naive recursive solutions can recompute the same values multiple times, leading to inefficient performance. (Confirmed)
    *   **Mitigation:** This inefficiency can often be mitigated using techniques such as **memoization** (caching results of expensive function calls and returning cached result when the same inputs occur again) or by converting the problem to an iterative solution using **dynamic programming**. (Confirmed)

**What is a Linked List?**

A **linked list** is a linear data structure that consists of a sequence of **nodes**, where each node contains data and a reference (or pointer) to the next node in the sequence. Unlike arrays, linked lists do not store elements in contiguous memory locations. Instead, they connect elements through these pointers, allowing for dynamic memory allocation and efficient insertion or deletion of elements. (Confirmed)

**Core Components:**

1.  **Node:** The fundamental building block of a linked list. Each node is an object or structure that typically contains two parts: (Confirmed)
    *   **Data:** The actual information or value that the node stores. (Confirmed)
    *   **Next Pointer/Reference:** A pointer or reference that stores the memory address of the next node in the sequence. If it's the last node, this pointer is usually `null` (or points to the head in a circular linked list). (Confirmed)

2.  **Head:** A pointer that points to the first node of the linked list. It is the entry point for accessing the list. If the list is empty, the head typically points to `null`. (Confirmed)

3.  **Tail (Optional but common):** A pointer that points to the last node of the linked list. Useful for efficient appending of new nodes to the end of the list. (Confirmed)

*   **Traversal:**
    1.  Start from the `head` of the list. (Confirmed)
    2.  Access the `data` of the current node. (Confirmed)
    3.  Move to the next node by following the `next pointer` of the current node. (Confirmed)
    4.  Repeat until the `next pointer` is `null` (or the `head` in a circular list). (Confirmed)

*   **Insertion (Adding a New Node):**
    *   **At the Beginning:**
        1.  Create a new node. (Confirmed)
        2.  Set the `new node's next pointer` to point to the current `head`. (Confirmed)
        3.  Update the `head` to point to the new node. (Confirmed)
    *   **At the End:**
        1.  Create a new node. (Confirmed)
        2.  Traverse the list until you reach the last node (whose `next pointer` is `null`). (Confirmed)
        3.  Set the `last node's next pointer` to point to the new node. (Confirmed)

**What is a Stack Data Structure?**

A **stack** is an abstract data type (ADT) that serves as a collection of elements, with the restriction that elements can only be added or removed from one end, referred to as the **top**. This restriction follows the **Last-In, First-Out (LIFO)** principle. Essentially, the last element added to the stack is the first one to be removed. (Confirmed)

**Core Principle: Last-In, First-Out (LIFO)**

Imagine a stack of plates:
*   You can only add a new plate to the top of the stack. (Confirmed)
*   You can only take a plate from the top of the stack. (Confirmed)
*   The plate you just put on top will be the first one you take off. (Confirmed)

**Fundamental Operations:**

These are the primary actions that can be performed on a stack:

1.  **Push:**
    *   **Action:** Adds an element to the top of the stack. (Confirmed)

**What is a Queue Data Structure?**

A **queue** is an **Abstract Data Type (ADT)** that represents a collection of elements, characterized by specific constraints on how elements can be added and removed. It operates on the **First-In, First-Out (FIFO)** principle, meaning the element that has been in the queue the longest is the next one to be removed. Elements are added exclusively at one end, known as the **rear** (or tail), and removed exclusively from the other end, known as the **front** (or head). This structure ensures an ordered and predictable processing sequence. (Confirmed)

**Core Principle: First-In, First-Out (FIFO)**

The FIFO principle is central to a queue's behavior and is analogous to a real-world waiting line:
*   **Entry Point (Rear):** New elements (like new customers) always join the queue at the very end. (Confirmed)
*   **Exit Point (Front):** Elements are always serviced or removed from the very beginning of the queue. (Confirmed)
*   **Order Preservation:** The first element to enter the queue will be the first element to exit, ensuring that elements are processed in the order they arrived. (Confirmed)
