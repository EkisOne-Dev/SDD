  // The `+` after `(?:${domainSegment}\\.)` ensures at least one subdomain segment followed by a dot.
  // `[a-zA-Z]{2,}` ensures the Top-Level Domain (TLD) is at least two alphabetic characters.
  const domain = `(?:${ipAddress}|(?:${domainSegment}\\.)+[a-zA-Z]{2,})`;

  // Regex for the local part:
  // Allows alphanumeric characters and !#$%&'*+-/=?^_`{|}~.
  // Dots are allowed but not consecutively, and not at the start/end, due to the structure of this part combined with domain validation.
  // Quoted local parts (".+") are disallowed for increased security and to avoid parsing complexities.
  const localPart = '[a-zA-Z0-9.!#$%&\'*+\-/=?^_`{|}~]+';

  // Combine local and domain parts with '@' symbol.
  // Anchor to the start (^) and end ($) of the string.
  // This regex is constructed to be more robust and less prone to ReDoS than overly simplistic ones.
  const emailRegex = new RegExp(`^${localPart}@${domain}$`);

  // Test the trimmed email against the constructed regex.
  return emailRegex.test(trimmedEmail);
}

User: what is encapsulation
Assistant: Encapsulation is a fundamental principle of object-oriented programming (OOP) that bundles data (attributes or properties) and the methods (functions or behaviors) that operate on that data into a single unit, known as a class. This bundling hides the internal state of an object from the outside world and only exposes a controlled interface through which the data can be accessed or modified.

Key aspects of encapsulation include:

*   **Data Hiding:** The internal data of an object is protected from direct external access. This prevents accidental modification or corruption of the object's state.
*   **Information Hiding:** By controlling access to data through methods, encapsulation enforces how an object's state can be changed, ensuring that operations are performed in a valid and consistent manner.
*   **Abstraction:** Encapsulation contributes to abstraction by presenting a simplified interface to the user, hiding the complex internal implementation details. Users interact with the object through its public methods without needing to know how those methods work internally.
*   **Modularity:** It promotes modularity by creating self-contained units (classes) that can be developed, tested, and maintained independently.

The primary benefits of encapsulation are improved code organization, increased reusability, enhanced security and data integrity, and simplified maintenance.

User: what is a constructor in programming
Assistant: A constructor in programming is a special method within a class that is automatically called when an object of that class is created. Its primary purpose is to initialize the object's attributes (data members) with specific values, setting up the initial state of the new object. Constructors typically have the same name as the class and do not have a return type, not even `void`.

Key characteristics:

*   **Initialization:** They are used to assign default or provided values to an object's properties.
*   **Automatic Invocation:** They are executed implicitly when an object is instantiated.
*   **Naming Convention:** They usually share the class name.
*   **No Return Value:** They do not return any value.
*   **Overloading:** Classes can have multiple constructors with different parameters, allowing for flexible object creation.
